Datepicker

<datepicker value="2017-03-01 12:10" mode="datetime"></datepicker>
Props
Name Type Default Description
name String The name attribute of the input element.
value String Initial value (date). Date needs to have a format such as 2017-03-01 or 2017-03-01 12:10
id String The id attribute of the input element.
placeholder String The placeholder attribute of the input element.
mode String date Mode can be set to date or datetime.
container-class String The class/classes of the container element.
days-of-week Array ['Su', 'Mo', 'Tu', ...] Days of the week.
months Array ['January', 'February', 'March', ...] Months.
icons Object { left: 'glyphicon glyphicon-chevron-left', ... } Icons.

    Tag input

    <tag-input value="Orange, Black" id="tag-input" placeholder="Add tag" :available-tags="availableTags" hidden-input-name="tags"></tag-input>
    Props
    Name Type Default Description
    available-tags Array An array with available tags ([{ id: 1, value: 'hey' }]).
    value [Array, String] An array or comma separated string with tags (['hey', 'there'] or hey, there).
    src String URL that will be used to fetch autocomplete suggestions (such as: /tag/search?q=).
    A query will be appended to this URL.
    There is no need to define available-tags when a remote source is defined.
    id String The id attribute of the input element.
    placeholder String The placeholder attribute of the input element.
    limit Number 0 Limit the number of autocomplete suggestions. No limit is applied by default.
    hidden-input-name String The name of a hidden input field that will be populated with tags (hey, there).
    Events
    Name Parameters Description
    input Tags (Array) Emitted on user input. Provides support for v-model.
    error Response (Object) Emitted when there was an error communicating with the remote source.

    Typeahead

    <typeahead initial-id="14" initial-value="John Stockton" id="typeahead-input" placeholder="Type a name" :initial-items="typeaheadItems" hidden-input-name="typeaheadId" @selected="onSelect"></typeahead>
    Props
    Name Type Default Description
    initial-items Array An array of items that will be used to display autocomplete suggestions ([{ id: 1, value: 'hey' }]).
    initial-value String Initial value of the autocomplete suggestion.
    initial-id String Initial id of the autocomplete suggestion.
    src String URL that will be used to fetch autocomplete suggestions (such as: /user/search?q=).
    A query will be appended to this URL.
    There is no need to define initial-items when a remote source is defined.
    id String The id attribute of the input element.
    placeholder String The placeholder attribute of the input element.
    limit Number 0 Limit the number of autocomplete suggestions. No limit is applied by default.
    name String The name of the input field.
    hidden-input-name String The name of a hidden input field that will be populated with autocomplete suggestion's id.
    Events
    Name Parameters Description
    selected Item (Object) Emitted when an item gets selected.
    error Response (Object) Emitted when there was an error communicating with the remote source.

    Ajax

    Using the Typeahead component with Ajax by defining the src prop will require 'vue-resource' peer dependency to be installed. Install 'vue-resource' with NPM and hook it to the main Vue instance. 

    import Vue from 'vue';
import VueResource from 'vue-resource';

Vue.use(VueResource);

    Customising templates

    The preferred way to customise templates is by using slots.

    Alternatively, you can use an inline template to fully replace the default template of any of the components.

    <component inline-template>
  <div>...your custom template goes here...</div>
</component>
