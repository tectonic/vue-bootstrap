Datepicker

<datepicker value="2019-08-31 23:55" mode="datetime"></datepicker>
Props
Name Type Default Description
name String The name attribute of the input element.
value String Initial value (date). Date needs to have a format such as 2017-03-01 or 2017-03-01 12:10
id String The id attribute of the input element.
container-class String The class/classes of the container element.
formatter Function null Callback used for formatting the date input. It receives current date in "YYYY-MM-DD (HH:MM)" format as a parameter. Can be easily used with momentum.js like this:
formatter(date) {
  return moment(date).locale(this.momentLocale).format(this.format);
}
use-am Boolean false If true then the hour will be displayed in AM/PM format.
days-of-week Array ['Su', 'Mo', 'Tu', ...] Days of the week.
highlight-today Boolean true Highlights today in the calendar.
icons Object { left: 'glyphicon glyphicon-chevron-left', ... } Icons for UI buttons.
mode String date Mode can be set to date, datetime or time.
months Array ['January', 'February', 'March', ...] Month names. Override it to use another language.
months-short Array ['Jan', 'Feb', 'Mar', ...] Short month names, used for the month selector view. Override it to use another language.
placeholder String The placeholder attribute of the input element.
translations Array { today: 'Go to today', clear: 'Clear selection', ... } Allows to override tooltips with another language.
disabled Boolean false Disables the input.
Events
Name Parameters Description
changed New date (String) Emitted when the date is changed.

    Popover

    Click me
    This is the content. 
    <popover title="Hello world!" placement="top" origin="left">
  <a class="btn btn-primary">Click me</a>
  <div slot="content">
    This is the content.
  </div>
</popover>
    Props
    Name Type Default Description
    title String Title.
    content String Content.
    A slot can also be used to insert content (like in the example above).
    placement String top Placement (top, bottom, left or right).
    origin String center

    Origin (top, bottom, left, right or center).

    It only makes sense to use certain combinations of placement and origin:

    • If the placement is top or bottom, the origin can be left, right or center.
    • If the placement is left or right, the origing can be top, bottom or center.

    In all other cases the origin will default to center.
    effect String fade Effect (currently only the fade effect is supported).
    trigger String click Trigger (click, hover or manual).
    append-to String body The element to which to append the popover.
    popover-class String Additional class for the popover.

    Name Type Default Description
    id String The id of the input element.

    value-property String value
    disabled Boolean false Disables the input.
    Events
    Name Parameters Description

    Tag input

    <tag-input value="Orange, Black" id="tag-input" placeholder="Add tag" :available-tags="availableTags" hidden-input-name="tags"></tag-input>
    Props
    Name Type Default Description
    available-tags Array An array with available tags ([{ id: 1, value: 'hey' }]).
    value [Array, String] An array or comma separated string with tags (['hey', 'there'] or hey, there).
    src String URL that will be used to fetch autocomplete suggestions (such as: /tag/search?q=).
    A query will be appended to this URL.
    There is no need to define available-tags when a remote source is defined.
    id String The id attribute of the input element.
    placeholder String The placeholder attribute of the input element.
    limit Number 0 Limit the number of autocomplete suggestions. No limit is applied by default.
    hidden-input-name String The name of a hidden input field that will be populated with tags (hey, there).
    Events
    Name Parameters Description
    input Tags (Array) Emitted on user input. Provides support for v-model.
    error Response (Object) Emitted when there was an error communicating with the remote source.

    Typeahead

    <typeahead initial-id="14" initial-value="John Stockton" id="typeahead-input" placeholder="Type a name" :initial-items="typeaheadItems" hidden-input-name="typeaheadId" @selected="onSelect"></typeahead>
    Props
    Name Type Default Description
    initial-items Array An array of items that will be used to display autocomplete suggestions ([{ id: 1, value: 'hey' }]).
    initial-value String Initial value of the autocomplete suggestion.
    initial-id String Initial id of the autocomplete suggestion.
    src String URL that will be used to fetch autocomplete suggestions (such as: /user/search?q=).
    A query will be appended to this URL.
    There is no need to define initial-items when a remote source is defined.
    id String The id attribute of the input element.
    placeholder String The placeholder attribute of the input element.
    limit Number 0 Limit the number of autocomplete suggestions. No limit is applied by default.
    name String The name of the input field.
    hidden-input-name String The name of a hidden input field that will be populated with autocomplete suggestion's id.
    Events
    Name Parameters Description
    selected Item (Object) Emitted when an item gets selected.
    error Response (Object) Emitted when there was an error communicating with the remote source.

    Ajax

    Using the Typeahead component with Ajax by defining the src prop will require the 'axios' peer dependency to be installed. Install 'axios' with NPM and hook it to the main Vue instance. 

    import Vue from 'vue';
import axios from 'axios'

Vue.prototype.$http = axios;

    Customising templates

    The preferred way to customise templates is by using slots.

    Alternatively, you can use an inline template to fully replace the default template of any of the components.

    <component inline-template>
  <div>...your custom template goes here...</div>
</component>
